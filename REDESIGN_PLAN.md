# tusi'm — план доработки: единая дизайн-система + «вау» + новые функции

Цель: привести все экраны (лендинг, кабинет организатора, гость, альбом, свайп, админка) к одной дизайн-системе, освежить и усилить визуал, добавить удобные функции. Параллельно — починить то, что сломается под RLS.

Легенда приоритета: **P0** — фундамент/блокеры, **P1** — основное, **P2** — приятные улучшения.

---

## Фаза 0. Дизайн-система — фундамент (P0)

Один источник правды, чтобы все экраны выглядели «из одной коробки». Сейчас стили продублированы в каждом файле (свои `.db-`, `.ev-`, `.cr-` классы) — отсюда расхождения.

**Токены** (вынести в `app/globals.css` как CSS-переменные):
- Цвета: фон `#09090b`, акцент `#C3073F`, плюс шкала прозрачных белых (текст/границы/поверхности), семантика success/warn/danger.
- Типографика: шкала размеров (display / h1 / h2 / body / caption), шрифты Unbounded (заголовки) + Onest (текст) уже есть.
- Скругления, тени, размытие (glass), отступы (4/8/12/16/24/32), z-index слои.
- Движение: единые easing/длительности, `prefers-reduced-motion`.

**Атмосфера (фишки, уже частично на лендинге — распространить аккуратно):**
- Аврора-свечение, плёночное зерно, виньетка — как фон-сигнатура бренда.
- Стеклянные панели (`backdrop-blur`) для шапок и модалок.
- Микро-взаимодействия: hover-подъём карточек, нажатия кнопок, плавные появления (reveal).

**Общие компоненты** (переиспользуемые, вместо копипасты):
- Кнопки (primary / secondary / ghost / danger), поля ввода, карточки, бейджи статуса, табы.
- Модалка/лайтбокс, тосты-уведомления (вместо `alert()`), пустые состояния, скелетоны загрузки.
- Шапка кабинета, нижняя навигация — единые.

## Фаза 2. Новые функции (P1, выбраны)

### A. Скачать все фото архивом (ZIP)
- Серверный роут `/api/organizer/download-zip?event_id=…` (проверка владения, service_role).
- Собирает все фото события в один ZIP, отдаёт потоком.
- Кнопка в кабинете события; учесть тариф (ZIP — от «Тусовки» и выше).
- На больших событиях — стриминговая упаковка, прогресс.

### B. Поиск / фильтр / сортировка + реальная статистика на дашборде
- Роут `/api/organizer/events` дополнить агрегатами: число фото, уникальных гостей на событие.
- Дашборд: поле поиска по названию, фильтр (все / активные / закрытые), сортировка (дата / активность).
- На карточке: «N фото · M гостей», статус, таймер.

## Идеи «что мог забыть» (P2 — выбрать, что зайдёт)

- **Обложка события**: загрузка картинки-обложки, брендирование страницы гостя (есть в тарифах).
- **Слайдшоу на экран**: полноэкранный режим для проектора на празднике (есть в топ-тарифах).
- **Избранное/лидерборд**: лучшие кадры по лайкам, «фото вечера».
- **Модерация**: организатор скрывает неудачные кадры до показа в общем альбоме.
- **Скачать одно фото / поделиться**: кнопка на каждом кадре, шеринг в соцсети/мессенджеры.
- **PWA / «на главный экран»**: гость добавляет ярлык, быстрый доступ без браузера.
- **QR-постер на стол**: красивый PDF/картинка «Сканируй и загружай».
- **Уведомление организатору** в Telegram, когда пошли первые загрузки.
- **Мультиязычность** страницы гостя (RU/EN) для интернациональных мероприятий.
- **Брендинг tusi'm/без него** по тарифу (watermark на бесплатном).

## Предлагаемый порядок работ

1. **Фаза 0** — дизайн-система (токены + базовые компоненты). Фундамент, без него полировка расползётся.
2. **Дашборд + кабинет события** — лицо для платящего клиента; сразу подключаем функции (статистика, поиск, ZIP).
3. **Гость + альбом + свайп** — виральная часть; чиним check-guest, освежаем.
4. **Админка** — переносим мутации на сервер, шлифуем.
5. **Лендинг + логин** — финальная косметика, всё уже на токенах.
6. Включение RLS + лимиты на сервере как завершение безопасности.
